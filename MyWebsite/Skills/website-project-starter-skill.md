# Skill: website-project-starter

## Purpose
Set up a new website project properly before design or coding begins.

## Inputs Required
- Business type
- Audience
- Website goal
- Pages or sections needed
- Visual direction
- Preferred stack

## Steps

### Step 1: Clarify the project
Ask concise questions to understand the business, audience, conversion goal, and desired style.

### Step 2: Define site architecture
Recommend pages and homepage section order.

### Step 3: Define visual direction
Summarize the site’s tone, motion style, and design references.

### Step 4: Define build constraints
List performance, SEO, accessibility, multilingual, and mobile requirements.

### Step 5: Generate core project files
Prepare:
- CLAUDE.md
- task_plan.md
- findings.md
- progress.md

## Quality Checks
- [ ] Site type is clear
- [ ] Audience is clear
- [ ] Conversion goal is clear
- [ ] Section structure is defined
- [ ] Constraints are listed

## Output
A proper kickoff structure and the four core project memory files.
